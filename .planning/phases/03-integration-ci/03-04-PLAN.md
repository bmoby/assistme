Create `.github/workflows/test.yml` with the following complete content:

```yaml
name: Tests

on:
  push:
    branches: ['**']
  pull_request:
    branches: [main]
  workflow_dispatch:

jobs:
  unit-tests:
    name: Unit Tests
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: '10'

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'

      - run: pnpm install --frozen-lockfile

      - name: Type check
        run: pnpm typecheck

      - name: Run unit tests
        run: pnpm test:unit

  integration-tests:
    name: Integration Tests
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: '10'

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'

      - uses: supabase/setup-cli@v1
        with:
          version: latest

      - name: Cache Supabase Docker images
        uses: actions/cache@v4
        with:
          path: ~/.supabase
          key: supabase-docker-${{ hashFiles('supabase/migrations/**') }}
          restore-keys: |
            supabase-docker-

      - run: pnpm install --frozen-lockfile

      - name: Start Supabase local
        run: supabase start

      - name: Run integration tests
        run: pnpm test:integration

  e2e-tests:
    name: E2E Tests (manual only)
    runs-on: ubuntu-latest
    if: github.event_name == 'workflow_dispatch'
    steps:
      - uses: actions/checkout@v4

      - name: Placeholder
        run: echo "E2E tests require Discord dev bot token. Configure in Phase 4."
```

**Key design decisions baked in:**
- `unit-tests` runs on EVERY push to ANY branch and on PRs. No Docker, no secrets, no Supabase. Includes `pnpm typecheck` before tests for early type error detection.
- `integration-tests` runs ONLY on pull_request to main. Uses `supabase/setup-cli@v1` to install Supabase CLI, `actions/cache@v4` to cache Docker images across runs, and `supabase start` to boot the local stack. `supabase db reset` is handled by `test/globalSetup.ts` (not in CI script) so the same lifecycle works locally and in CI.
- `e2e-tests` runs ONLY on `workflow_dispatch` (manual trigger). Currently a placeholder — Phase 4 will add real Discord E2E tests.
- `pnpm install --frozen-lockfile` is the correct CI flag (not `--ci` which is npm-only).
- The `pnpm/action-setup@v4` version field uses string `'10'` to match the project's pnpm major version.
- The Supabase Docker cache key includes a hash of all migration files — cache invalidates when schema changes.
- `supabase start` runs WITHOUT a separate `supabase db reset` step because `test/globalSetup.ts` already handles `db reset` before tests. Running it twice would be redundant.

**Do NOT:**
- Add any Discord secrets to the workflow (those are for Phase 4 E2E only)
- Add coverage reporting to CI yet (coverage thresholds are enforced locally via `pnpm test:coverage`)
- Add any `workflow_dispatch` inputs to the e2e-tests job (Phase 4 will define these)

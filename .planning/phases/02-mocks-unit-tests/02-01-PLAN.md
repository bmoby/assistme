**dm-handler.ts** -- Add at the very bottom of the file, after the `setupDmHandler` export:

```typescript
/**
 * Test-only: reset module-level state between tests.
 * Prevents conversation state and processing locks from leaking across test runs.
 */
export function _clearStateForTesting(): void {
  if (process.env['NODE_ENV'] !== 'test') return;
  conversations.clear();
  processingLocks.clear();
}
```

The `conversations` Map is defined on line 23, `processingLocks` Map on line 26. Both are module-level `const` Maps that accumulate state across calls. Without clearing them, test A's conversation state leaks into test B when tests run in the same Vitest worker (Pitfall 3 from research).

The `discordClient` variable (line 9) does NOT need clearing here -- it gets reassigned each time `setupDmHandler(mockClient)` is called in `beforeEach` (Pitfall 5 prevention).

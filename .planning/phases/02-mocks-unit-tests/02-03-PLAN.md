DM handler internal behavior (from source read):
- `setupDmHandler(client)` stores client ref in module-level `discordClient`
- Registers `client.on('messageCreate', ...)` handler
- Handler checks: not bot, `!message.guild` (is DM), has student in DB
- Queues messages per user via `processingLocks` Map (prevents concurrent processing)
- Calls `runDmAgent(context)` with conversation history
- If `result.submissionId` is set, calls `notifyAdminChannel(...)` using `discordClient.channels.cache.get(CHANNELS.admin_channel_id)`

Admin handler internal behavior:
- `setupAdminHandler(client)` registers `client.on('messageCreate', ...)` handler
- Handler checks: not bot, message in admin channel, user is admin (role 'tsarag')
- Manages conversation state per admin user
- Calls `runTsaragAgent(context)` with conversation history and Discord action callbacks
- Pending action confirmation flow (agent proposes, admin confirms)

Review-buttons internal behavior:
- `registerReviewButtons()` calls `registerButton(prefix, handler)` for 4 prefixes
- Each button handler: parses exercise ID from `interaction.customId`, calls `deferReply`, fetches exercise/session, performs status update, calls `editReply`

## Objectif du module

L'etudiant comprend le digital, sait decomposer un systeme, et decouvre les outils IA. Il passe de "consommateur de tech" a "createur de tech". 2 choses en ligne a la fin du module.

**Bloom** : Comprendre + Appliquer (basique)
**Scaffolding** : Guide (tout est montre, l'etudiant reproduit)

## Principes cles pour ce module

### 1. Premiere semaine = la plus critique
- 73% des abandons se decident dans les 7-14 premiers jours (Kizilcec, 2013)
- Le "modele mental de soi-en-tant-qu'apprenant" se forme dans la premiere semaine
- Quick Win = investissement motivationnel le plus rentable de toute la formation

### 2. Analogies avant tout
- Le cerveau apprend en greffant le nouveau sur l'existant (Hofstadter)
- L'analogie restaurant est universelle — tout le monde a ete au restaurant
- Cycle en 3 temps : introduction par analogie → construction → rupture explicite ("ou l'analogie casse")
- UNE analogie bien developpee > trois analogies superficielles

### 3. Spacing effect
- Les concepts de S2 sont REVUS en S3 (meme app, schema enrichi = rappel a J+5)
- Retrieval quiz de 3 questions au debut de chaque session (sauf S1)
- L'etudiant RECUPERE l'info de memoire — pas le formateur qui resume

### 4. Concret avant abstrait
- JAMAIS de theorie sans experience prealable
- L'etudiant deploie AVANT de comprendre comment ca marche (S1)
- L'etudiant dessine AVANT de definir les termes (S2)

### 5. Barrieres cognitives a gerer
- Identite negative ("la tech c'est pas pour moi") → Quick Win casse cette croyance
- Charge cognitive cumulative → max 35 min de nouveau contenu, puis pause active
- Langage exclusif → bannir le jargon, utiliser les mots de l'analogie

## Insights des recherches existantes (rapports A, B, C)

### De la Recherche A (psychologie)
- **5 barrieres cognitives** : abstraction non ancree, charge cognitive cumulative, absence de feedback visible, langage exclusif, identite negative
- **Syndrome de l'imposteur de domaine** : "je viens d'un monde different" — plus virulent que l'imposteur classique
- **Declencheurs dans la formation** : quand quelqu'un pose une question qu'on ne comprend meme pas, quand l'IA produit quelque chose qu'on ne sait pas evaluer
- **Strategie cle** : "Droit a la question stupide" institutionnalise des S1

### De la Recherche B (pedagogie)
- **4 niveaux de competence** : litteratie numerique → comprehension computationnelle → creation assistee → dev technique. On vise niveaux 2-3.
- **Computational thinking** : decomposition, patterns, abstraction, algorithmes — tout peut s'enseigner sans code
- **Analogie restaurant** : couvre toute l'architecture web en une analogie — la plus complete et la plus universelle
- **Cycle en 3 temps** pour les analogies : introduction → construction → rupture
- **PBL (Project-Based Learning)** : effect size d=0.71 en faveur du PBL vs theorie-first pour retention long terme

### De la Recherche C (structure)
- **Premiere semaine** : le "Premier Succes Rapide" est la fondation de tout
- **Pre-boarding J-7** : message de bienvenue, instructions Discord, action immediate (se presenter)
- **Retrieval practice** : l'acte de se souvenir > l'acte de relire (retention x1.5 a x2)
- **Attention soutenue** : degradation apres 45-50 min → decoupe en blocs de 25-45 min avec pause active
- **Contrat d'engagement public** : "Je m'engage a completer les 12 semaines"

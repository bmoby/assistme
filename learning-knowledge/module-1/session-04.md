## Философия С4

**С4 убирает страх перед кодом.** Студенты уже создали HTML-страницы, но код для них остаётся чёрным ящиком. Эта сессия разрушает три мифа:

1. **"Если я трону файл, всё сломается"** — нет. Файлы и папки — это как на рабочем столе. Ничего хрупкого.
2. **"Код — это магия для программистов"** — нет. Код — это инструкции, которые ждут вашего действия. Как законы: написаны заранее, применяются по ситуации.
3. **"Я не понимаю, что происходит, когда я открываю сайт"** — после этой сессии вы будете точно знать.

**Центральная идея** : программирование — это предвидение. Разработчик предусматривает действия заранее. Код ждёт. Пользователь действует. Код исполняется.

**Зачем сейчас** : всё, что последует в формации, основано на программировании. JavaScript — наш язык. Прежде чем его изучать, нужно понять его роль и увидеть, как работает веб изнутри.

## ЧАСТЬ 1 — Структура проекта (15 мин)

### Папки — это просто папки

На вашем компьютере у вас есть папка "Фотографии". Внутри — "Семейные", "Путешествия", "Работа". В каждой — файлы. Вы не боитесь перемещать фотографии из одной папки в другую. Вы не боитесь создать новую папку.

**Проект с кодом — это то же самое.** Дossiers, файлы. Ничего хрупкого. Если вы создадите новую папку или переместите файл — ничего не взорвётся.

> "Код — это не бомба. Это текст в файлах. Файлы в папках. Точно как на вашем рабочем столе."

### Что такое слеши

**Демо в прямом эфире :** открыть Finder (Mac) / Проводник (Windows) и показать путь к файлу.

Путь к файлу на компьютере :

```
/Users/maga/Documents/mon-site/page-accueil/index.html
```

Каждый `/` (слеш) означает: **войти внутрь этой папки**.

- `Users` → папка пользователей
- `maga` → моя папка
- `Documents` → документы
- `mon-site` → мой проект
- `page-accueil` → папка для главной страницы
- `index.html` → сам файл

**Показать в навигаторе :** открыть страницу из проекта С3, посмотреть адресную строку :

```
file:///Users/maga/Documents/mon-site/page-accueil/index.html
```

Те же слеши. Тот же принцип. Навигатор просто показывает путь к файлу.

### Вспомним С3

Ваш проект из С3 :

```
mon-site/
├── page-accueil/
│   └── index.html
├── page-a-propos/
│   └── index.html
└── page-contact/
    └── index.html
```

Вы **уже** организовали проект по папкам. HTML-файлы — в соответствующих папках. Это и есть структура проекта. В реальных проектах есть также папки для CSS, для JavaScript, для картинок — но принцип тот же: разделять вещи по категориям.

> "Когда вы разбираете шкаф — вы складываете вещи по полкам. Проект с кодом — то же самое."

## ЧАСТЬ 2 — Демо: сила кода (10 мин)

### Код делает то, что человеку не под силу

**Демо на экране Магомеда** (студенты смотрят, не устанавливают ничего) :

На экране — папка со 100 файлами с беспорядочными именами :

```
IMG_3841.jpg
photo_2024_final_v2.jpg
DSC00234.jpg
screenshot_old.jpg
...
```

Магомед запускает скрипт из 10 строк. За 1 секунду все файлы переименованы :

```
thailande-001.jpg
thailande-002.jpg
thailande-003.jpg
...
```

**Реакция** : на это ушла бы минимум 20 минут вручную, и можно было бы ошибиться. Код сделал это за секунду, без единой ошибки.

> "Вот что умеет код. Давать инструкции компьютеру, и он делает за вас. Быстрее, точнее, без усталости."

### Связка с JavaScript

Этот скрипт — это JavaScript, запущенный напрямую на компьютере (не в навигаторе). JavaScript — универсальный язык: он работает и в навигаторе, и вне него.

Но прежде чем учить **как** писать JavaScript, нужно понять **зачем** он существует и **что он делает** в навигаторе. Именно это мы сейчас разберём.

## ЧАСТЬ 3 — Зачем JavaScript (25 мин)

### Три языка веба — напоминание за 1 минуту

| Язык       | Роль        | Аналогия                                |
| ---------- | ----------- | --------------------------------------- |
| HTML       | Структура   | Кирпичи, стены, фундамент               |
| CSS        | Внешний вид | Краска, обои, декор                     |
| JavaScript | Поведение   | Электричество — выключатели, свет, лифт |

Без JavaScript сайт — это афиша. Красивая, но статичная. JavaScript делает сайт **живым**.

### Код не выполняется, когда его читают

Это ключевой концепт. Когда навигатор получает JavaScript, он его **читает** и **запоминает**. Но не выполняет сразу.

**Аналогия с законами :**

В каждой стране есть законы. Они написаны и ждут. Закон о краже существует, но он применяется **только когда кто-то украл**. Если никто не ворует — закон существует, но никогда не выполняется.

JavaScript — то же самое:

```
Если пользователь нажмёт на кнопку "Отправить"
  → Отправить данные на сервер
  → Показать "Отправлено!"
```

Этот код прочитан навигатором. Навигатор знает, что делать. Но пока никто не нажал кнопку — ничего не произойдёт. Код **ждёт**.

> "Программирование — это предвидение. Разработчик продумывает заранее все возможные действия пользователя. Код ждёт. Пользователь действует. Код исполняется."

### Разработчик предвидит всё заранее

Каждое действие, которое вы можете совершить на сайте — кликнуть, прокрутить, написать текст, нажать Enter — **предусмотрено разработчиком**.

Если кнопка ничего не делает — значит разработчик не предусмотрел действие для этой кнопки. Не баг навигатора. Не проблема компьютера. Просто никто не написал: "когда нажмут на эту кнопку, сделай вот это."

Примеры :

- Нажал "Добавить в корзину" → JavaScript обновляет счётчик товаров
- Навёл мышку на аватар → JavaScript показывает карточку профиля
- Прокрутил страницу вниз → JavaScript загружает новые посты (Instagram)
- Написал в поиск → JavaScript показывает подсказки

Всё это — заранее предусмотренные инструкции.

### Сайт — это приложение, которое вы скачиваете каждый раз

Когда вы установили Instagram на телефон — вы скачали его один раз. После этого он на вашем телефоне.

Сайт работает **иначе**. Каждый раз, когда вы вводите адрес и нажимаете Enter :

1. Ваш навигатор **отправляет запрос** к серверу (кухне)
2. Сервер **отвечает** : отправляет HTML, CSS, JavaScript
3. Навигатор **получает** эти файлы и отображает страницу
4. Вы видите сайт

Если вы отключите интернет и обновите страницу — ничего не появится. Потому что файлы не сохраняются у вас — навигатор каждый раз скачивает их заново.

> "Каждый раз, когда вы открываете сайт, вы фактически скачиваете маленькое приложение. Только в отличие от мобильного приложения, навигатор его забывает."

### Что происходит после загрузки

Итак, вы скачали сайт. На вашем компьютере теперь есть HTML (структура), CSS (стиль) и JavaScript (поведение). Навигатор всё прочитал. JavaScript — **на вашем компьютере**, в вашем навигаторе.

Теперь :

- HTML отображается → вы видите страницу
- CSS применён → страница красивая
- JavaScript прочитан → **и ждёт ваших действий**

Когда вы нажимаете на кнопку, это **ваш компьютер** исполняет JavaScript. Не сервер. Не облако. Ваш компьютер.

А если JavaScript нужны данные, которых у него нет (например, список товаров из базы данных) — он **отправляет запрос** к серверу, получает данные, и отображает их на странице. Это вторая волна запросов — уже не для скачивания сайта, а для получения информации.

## ЧАСТЬ 4 — Веб изнутри: DevTools (25 мин)

### 4.1 — Инспектор элементов (10 мин)

**Все делают одновременно :**

1. Откройте **Instagram** (или любой сайт) в навигаторе
2. Правый клик на любом элементе → **"Просмотреть код"** / **"Inspect"**
3. Справа или снизу появилась панель — это **DevTools** (инструменты разработчика)

Что вы видите :

- **Вкладка Elements** — весь HTML-код страницы. Да, Instagram — это HTML. Точно как ваши страницы из С3, но сложнее
- Стили справа — CSS. Цвета, отступы, шрифты
- Всё, что вы видите на экране — это результат этого кода

**Эксперимент** : найдите текст на странице (например, заголовок), дважды кликните на него в панели Elements, измените текст. Заголовок изменился на экране.

> "Вы только что модифицировали Instagram. На вашем экране. Это не магия — это просто HTML, который ваш навигатор показывает. Вы изменили копию, которая на вашем компьютере."

При обновлении страницы — изменение исчезнет. Потому что навигатор **заново скачает** оригинальные файлы с сервера.

### 4.2 — Вкладка Network: видим запросы (15 мин)

**Все делают одновременно :**

1. В DevTools переключитесь на вкладку **Network**
2. Обновите страницу (`Cmd+R` / `Ctrl+R`)

Что вы видите : **десятки строк появляются**. Каждая строка — это один запрос. Ваш навигатор запросил файлы у сервера, и каждая строка — один полученный файл.

Показать :

- Файлы HTML — структура
- Файлы CSS — стили
- Файлы JS — код
- Картинки — изображения, которые вы видите на странице

**Эксперимент** : не обновляя страницу, прокрутите ленту Instagram вниз. Новые строки появляются в Network. Это JavaScript отправляет запросы за новыми постами.

> "Когда вы скроллите Instagram и появляются новые фотографии — это именно то, о чём мы говорили. JavaScript, который на вашем компьютере, видит, что вы прокрутили вниз. Разработчик предусмотрел это. И JavaScript отправляет запрос к серверу: 'дай мне ещё 10 постов'. Сервер отвечает. JavaScript получает данные и размещает их на странице."

### Доказательство всего, что мы обсудили

Вкладка Network — это **доказательство** всей теории :

- Обновили страницу → скачали приложение (первая волна запросов)
- Нажали кнопку или прокрутили → JavaScript запросил данные (вторая волна)
- Отключите интернет, обновите → ничего. Подтверждено: сайт не хранится у вас

## ЧАСТЬ 6 — Полная картина + тизер С5 (10 мин)

### Соберём всё вместе

На С2 мы узнали модель: Клиент / API / Сервер / База данных.
Сегодня мы увидели это **вживую** :

```
ВАШ НАВИГАТОР (клиент)                    СЕРВЕР (кухня)
┌─────────────────────┐                   ┌─────────────────┐
│  HTML — структура   │ ◄── запрос 1 ──► │  Хранит файлы   │
│  CSS  — стиль       │    (скачать сайт) │  HTML/CSS/JS    │
│  JS   — поведение   │                   │                 │
│                     │ ◄── запрос 2 ──► │  Хранит данные   │
│  JS ждёт действий   │    (получить     │  (база данных)  │
│  Вы нажали кнопку → │     данные)       │                 │
│  JS выполнился      │                   │                 │
└─────────────────────┘                   └─────────────────┘
```

1. Вы вводите адрес → навигатор скачивает HTML/CSS/JS с сервера
2. Навигатор отображает страницу и читает JavaScript
3. JavaScript **ждёт** ваших действий
4. Вы действуете → JavaScript исполняется на **вашем** компьютере
5. Если нужны данные → JavaScript отправляет запрос к серверу → получает → отображает

Всё. Это вся схема работы **любого** сайта в мире.

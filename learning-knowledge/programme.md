# Формация "Pilote Neuro" Сессия 2 — Программа

> 12 недель, 24 сессии, 6 модулей

**Цель** : Научить нетехнарей проектировать любую цифровую систему с помощью ИИ.
**Формат** : 3 месяца, 2 сессии по 2ч/неделю + задания + документы

---

## Видение

Формация НЕ учит программированию. Она передает:

1. **Понимание** — Как работает код (не синтаксис, а логика)
2. **Архитектура** — Как проект декомпозируется и структурируется
3. **Использование ИИ** — Как общаться с Claude, ChatGPT и др. для создания продуктов
4. **Критическое мышление** — Проверять, тестировать, ставить под вопрос работу ИИ
5. **Дизайн и эргономика** — Объективные правила (mobile-first, UX)
6. **Mindset** — ИИ ускоряет, но не упрощает. Когда все быстрые, нужно делать ЛУЧШЕ.

---

## Педагогические принципы

1. **Конкретное перед абстрактным** — Сначала проблема, потом метод как решение.
2. **Spec-Driven Development** — Спек = источник истины. Преподается ПОСЛЕ пережитого хаоса.
3. **Проект — красная нить** — Каждый строит СВОЙ проект. Вся теория служит проекту.
4. **Just-in-time tooling** — Каждый инструмент вводится в момент, когда он нужен.
5. **Человек делает то, что ИИ не может** — Env vars, API keys, регистрации, валидация.
6. **Последовательный scaffolding** — С проводником (М1) -> полуоткрытый (М2-М3) -> свободный (М4+).
7. **Flipped classroom** — Видео (10-15 мин) ДО сессии. Лайв = 100% практика.
8. **Peer review спеков** — Оценивать спеки других, находить слабости, предлагать альтернативы.
9. **Артефакт на каждой сессии** — URL, схема, спек, прототип. Никогда чисто теории.

| Время | Блок | Содержание |
|-------|------|------------|

## Система анти-отчисления

4 критических момента:
1. **Недели 1-2** : Технические проблемы, изоляция -> Крепкий пре-бординг + контакт в первые 72ч
2. **Недели 4-5** : Долина отчаяния -> Церемония середины, легкое задание
3. **Недели 7-8** : Усталость -> Облегченная нагрузка
4. **Недели 10-11** : Стена проекта -> Обязательные контрольные точки

**Правило** : 2 красных показателя = контакт 48ч. 3+ = личный звонок.

## Программа — 6 модулей, 24 сессии

### Нить прогрессии

```
Недели 1-2  : ОТКРЫТЬ            -> "Я понимаю цифровой мир и умею пользоваться ИИ"
Недели 3-4  : МЕТОД              -> 1ч хаоса -> откровение -> спек -> rebuild
Недели 5-6  : АРСЕНАЛ ИИ        -> Config, skills, agents
Недели 7-8  : СТРОИТЬ            -> Спринты со спеком + полный арсенал
Недели 9-10 : ПРОФЕССИОНАЛИЗАЦИЯ -> Предложение клиенту, peer review, MVP
Недели 11-12: СДАТЬ              -> Тесты, деплой, demo day
```

---

### МОДУЛЬ 1 — Открыть (Недели 1-2)

Студент понимает цифровой мир, умеет декомпозировать, знает инструменты ИИ.

| Сессия | Тема | Артефакт |
|--------|------|----------|
| С1 | Кик-офф + онбординг | Discord настроен, расписание выбрано |
| С2 | Цифровой мир: зал и кухня | Схема архитектуры приложения |
| С3 | Цифровой мир: холодильник, поставщики, охранник | Полная схема архитектуры |
| С4 | Ландшафт ИИ 2026 | Страница в ИИ-редакторе |

---

### МОДУЛЬ 2 — Метод (Недели 3-4)

Студент проживает хаос, открывает spec-driven метод, пишет спек, получает peer review.

| Сессия | Тема | Артефакт |
|--------|------|----------|
| С5 | Контролируемый хаос | Кривой прототип + список фрустраций |
| С6 | Откровение: метод в 6 шагов | Шаги 1-3 метода на своем проекте |
| С7 | Написать спек + первый конфиг | Полный спек + CLAUDE.md/.cursorrules |
| С8 | Управлять ИИ + peer review | Задачи по методу + спек улучшен пирами |

Метод в 6 шагов: ОПИСАТЬ -> ИССЛЕДОВАТЬ -> СПРАШИВАТЬ -> ПОНЯТЬ -> ПРИОРИТИЗИРОВАТЬ -> СПЕЦИФИЦИРОВАТЬ

---

### МОДУЛЬ 3 — Арсенал ИИ (Недели 5-6)

Студент настраивает ИИ-инфраструктуру (skills, agents) и продвигает проект. Церемония середины.

| Сессия | Тема | Артефакт |
|--------|------|----------|
| С9 | Skills и переиспользуемые промпты | 4 скилла настроены и протестированы |
| С10 | Агенты и спринт | 1 персональный агент + прогресс проекта |
| С11 | Углубленный спринт | Проект с auth и/или интеграцией |
| С12 | Церемония середины + peer review | Улучшенный проект + обновленная мотивация |

4 основных скилла: Explainer, UX Review, Spec Sync, Pre-commit Review

---

### МОДУЛЬ 4 — Строить (Недели 7-8)

Структурированные спринты с растущей автономией. Дебаг и критическое мышление.

| Сессия | Тема | Артефакт |
|--------|------|----------|
| С13 | Спринт: интерфейс + данные | Front-end + БД подключены, CRUD работает |
| С14 | Спринт: продвинутые фичи | Проект обогащен фичами |
| С15 | Дебаг и критическое мышление | Найденные баги + чеклист ревью |
| С16 | Peer review спеки vs продукт | Проект улучшен фидбэком |

4 вопроса ревью: Неожиданное? Сеть? Безопасность? Одна вещь или много?

---

### МОДУЛЬ 5 — Профессионализация (Недели 9-10)

Метод в профессиональном контексте. Продвинутый peer review. Полная автономия. MVP.

| Сессия | Тема | Артефакт |
|--------|------|----------|
| С17 | Предложение клиенту | Структурированное предложение со спеком |
| С18 | Автономный спринт | Прогресс проекта |
| С19 | Продвинутый peer review спеков | Улучшенный спек + альтернативы |
| С20 | MVP Checkpoint | Рабочий MVP + план на 2 недели |

---

### МОДУЛЬ 6 — Сдать (Недели 11-12)

Отполировать, протестировать, задеплоить, представить.

| Сессия | Тема | Артефакт |
|--------|------|----------|
| С21 | Тестирование реальным пользователем | Отзывы + список исправлений |
| С22 | Продакшн-деплой | Проект онлайн, URL доступен |
| С23 | Репетиция Demo Day | Отрепетированная презентация |
| С24 | DEMO DAY + Выпуск | Готовый продукт + роль Discord |

Финал: студент умеет специфицировать, оснастить свой ИИ, строить методично и сдать реальный продукт.

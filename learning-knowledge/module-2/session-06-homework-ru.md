# Домашнее задание после сессии 6

## Что нужно сделать

До следующей сессии вам нужно сделать **личный front-end проект на Next.js**.

Это не просто красивый сайт.
Это маленькое приложение, которое помогает человеку:

- понять ваше предложение
- посмотреть варианты
- ответить на несколько вопросов
- получить рекомендацию
- оставить заявку

---

## Какая тема проекта подходит

Проект должен быть связан с вашей реальной деятельностью.

Подходящие темы:

- коучинг
- консультации
- агентство
- фриланс
- обучение
- недвижимость
- спорт, здоровье, питание
- локальные услуги

Главное правило:

> проект должен помогать пользователю принять решение

---

## Что обязательно должно быть в проекте

### 1. Несколько страниц

У вас должно быть **несколько настоящих страниц**.

Например:

- главная
- услуги или предложения
- мини-опрос
- результат
- контакт

Точное количество страниц вы выбираете сами.
Но это должен быть не один экран со scroll, а настоящий сайт с навигацией.

### 2. Настоящая навигация

Пользователь должен уметь переходить между страницами.

Например:

- с главной на услуги
- с услуг на мини-опрос
- с мини-опроса на результат
- с результата на контакт

### 3. Несколько переиспользуемых компонентов

У вас должно быть **несколько компонентов**, которые используются в разных местах проекта.

Примеры:

- меню
- карточка услуги
- блок с кнопкой
- шаг вопроса
- progress bar
- блок результата

Не копируйте один и тот же код много раз.

### 4. Одна умная функция для списка результатов

Нужна **минимум одна** полезная функция:

- сортировка по цене
- сортировка по дате
- сортировка по уровню
- сортировка по другому понятному критерию

Этого достаточно.
Не надо делать слишком много логики сразу.

### 5. Сбор информации и рекомендация

Пользователь должен ответить на несколько вопросов.
После этого сайт должен показать **рекомендацию**.

Это может быть:

- подходящая услуга
- подходящий пакет
- подходящий формат
- простой результат

Рекомендация должна зависеть от ответов пользователя.

### 6. Local Storage

Мы уже говорили, что браузер может **временно хранить информацию**.

Это называется:

`Local Storage`

Используйте его как маленькую временную базу данных в браузере.

Например, можно сохранить:

- ответы пользователя
- выбранный вариант
- результат мини-опроса

Сейчас **не нужно** делать:

## Как работать с ИИ

Нет одного обязательного порядка.

Если вы делаете проект с помощью Antigravity, Codex или другого ИИ, работайте так:

### 1. Сначала объясните ИИ, что вы хотите сделать

Напишите коротко:

- тема проекта
- для кого этот сайт
- какие страницы нужны
- какая будет сортировка
- какой будет мини-опрос
- какую рекомендацию должен показать сайт

### 2. Просите ИИ делать одну задачу за раз

Не просите:

> сделай мне сразу весь проект

Лучше просите маленькими частями:

- создай страницы
- сделай навигацию
- вынеси повторяющиеся блоки в компоненты
- добавь сортировку
- сделай мини-опрос
- покажи рекомендацию
- сохрани данные в `Local Storage`

### 3. После каждого изменения проверяйте проект локально

Проверяйте:

- открываются ли страницы
- работает ли навигация
- работает ли сортировка
- сохраняются ли ответы
- появляется ли рекомендация

### 4. Если что-то сломалось, исправляйте одну проблему за раз

Скопируйте точную ошибку и отправьте ее ИИ.

Не меняйте сразу весь проект.

### 5. Работайте просто

Вам не нужно делать все идеально.

Главное:

- несколько страниц
- навигация
- несколько компонентов
- одна сортировка
- мини-опрос
- рекомендация
- `Local Storage`
- страница контакта или заявки

До следующей сессии подготовьте:

2. короткий список функций проекта
3. скриншот или короткое видео, где видно страницы, сортировку, мини-опрос и рекомендацию

Проект можно оставить только у себя на компьютере.

---

## Если вы застряли

Не паникуйте.

Делайте так:

1. прочитайте точную ошибку
2. скопируйте точную ошибку
3. меняйте только одну вещь за раз
4. просите ИИ исправить маленькую конкретную проблему
5. снова проверьте проект

Правильный запрос к ИИ:

> У меня ошибка: [вставьте точный текст ошибки]. Исправь минимально. Не меняй весь проект.

Неправильный запрос:

> Ничего не работает. Исправь все.

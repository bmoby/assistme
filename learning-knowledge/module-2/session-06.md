## Философия С6

**С6 — это не сессия про новую теорию. С6 нужна для того, чтобы превратить хрупкие действия в уверенные рефлексы.**

На С5 студенты увидели :

1. что современный проект — это уже не просто файл `index.html`
2. что локальный сервер может работать прямо на их компьютере
3. что Next.js дает структуру настоящему веб-проекту
4. что приложение может жить локально, а потом быть выложено в интернет

Теперь нужно сделать педагогически очень важную вещь :

- спокойно повторить основные действия
- повторить их несколько раз
- дать проявиться нормальным ошибкам
- научиться исправлять их без паники

### Что важно сказать в самом начале

- вы уже очень многое поняли за первые сессии
- то, что вы изучили, станет намного яснее, когда вы начнете трогать это руками
- практика даст вам 4 вещи :
  - лучшее понимание
  - первые признаки настоящего владения
  - понимание собственных границ
  - более быстрое понимание следующих тем

Но нужно честно сказать и другое :

- искусственный интеллект не всегда будет точно вас слушать
- иногда он создаст не то, что вы просили
- ошибки будут
- эти ошибки будут зависеть от вашего компьютера, установки и способа работы

**Это нельзя подавать как мелочь.**
Наоборот, нужно прямо сказать :

> "Одна из целей этой сессии — не только практиковаться. Одна из целей — научиться спокойно чинить то, что ломается."

### Мостик к JavaScript

Нужно сделать очень простое напоминание :

> "Next.js не убирает JavaScript. Next.js организует JavaScript."

Иначе говоря :

- страницы, которые мы создаем, все равно остаются кодом
- компоненты, которые мы создаем, все равно остаются кодом
- навигация между страницами — это тоже заранее предусмотренное поведение
- приложение работает только потому, что код был написан, сохранен и выполнен

То есть **никакой новой магической вселенной не появилось**.

Мир все тот же :

- HTML = то, что отображается
- CSS = то, как это выглядит
- JavaScript = логика и поведение

Next.js просто добавляет :

- более крепкую структуру
- соглашения
- специальные файлы
- более аккуратный способ запускать и организовывать приложение

**Центральная идея С6** :

> "Сегодня вы почти не изучаете новую теорию. Сегодня вы учитесь без страха управлять настоящим проектом."

**Скрытая цель С6** :

> убрать страх "я все сломаю" и заменить его на рутину : "я читаю, проверяю, исправляю, перезапускаю".

## ЧАСТЬ 1 — Почему JavaScript по-прежнему нужен в Next.js (10 мин)

### 2.1 — Нельзя оставлять впечатление, что Next.js заменил базу

Если это не объяснить, студенты могут начать думать :

- "мы закончили с JavaScript"
- "теперь Next.js все делает сам"
- "мне уже не нужно понимать, что я пишу"

Это нужно исправить сразу.

### 2.2 — Очень простое объяснение

Нужно сказать :

> "Когда вы создаете страницу в Next.js, вы все равно пишете инструкции. Когда вы создаете компонент, вы все равно пишете инструкции. Когда вы нажимаете на ссылку, навигация тоже заранее предусмотрена в коде."

Значит :

- С4 показала внутреннюю логику веба и JavaScript
- С5 показала современный проект и локальный сервер
- С6 переводит все это в практику внутри реальной структуры

### 2.3 — Два сверхпростых определения

**Страница** :

> экран, связанный с конкретным адресом

Примеры :

- `/` = главная страница
- `/offers` = страница с предложениями
- `/contact` = страница контакта

**Компонент** :

> переиспользуемый блок, который можно поставить на разные страницы

Примеры :

- карточка услуги
- меню
- кнопка
- блок с отзывом

### 2.4 — Формула для лайва

> "Страница = место назначения. Компонент = переиспользуемая деталь."

### 2.5 — Фраза, которую нужно повторять

> "Next.js организует проект. Но это все равно код."

## ЧАСТЬ 2 — Демо : создать проект Next.js, включить, выключить и снова включить (22 мин)

### 3.1 — Главная цель демо

Сегодня главное — не дизайн.
Главное сегодня :

> научиться запускать проект и останавливать его без страха.

### 3.2 — Один путь для всей когорты

Чтобы не запутывать людей, нужно сказать прямо :

> "Сегодня мы не смешиваем инструменты. Для всей группы мы идем по одному и тому же пути."

Не нужно открывать войну :

- `npm`
- `pnpm`
- `yarn`
- `bun`

Для этой сессии достаточно одного пути.

### 3.3 — Создание проекта

Можно создать, например, проект с именем :

```bash
npx create-next-app@latest pilote-business-hub
```

Потом :

```bash
cd pilote-business-hub
```

Если инструмент задает вопросы, можно просто сказать :

> "Берите рекомендованные опции."

Педагогическая цель сейчас не в том, чтобы разбирать каждую опцию.
Педагогическая цель :

- создать настоящий проект
- не потеряться
- как можно быстрее добраться до работающего приложения

### 3.4 — Включить проект

Потом запустить :

```bash
npm run dev
```

Сразу сказать :

> "Запомните только это : `npm run dev` — это команда, которая включает проект в режиме разработки."

Когда сервер уже работает, открыть :

```text
http://localhost:3000
```

И еще раз проговорить :

- `localhost` = ваш компьютер
- `3000` = порт, который используется здесь
- пока терминал работает, приложение отвечает

### 3.5 — Главная ментальная точка

> "Ваш браузер больше не просто читает файл. Он разговаривает с приложением, которое запущено на вашей машине."

### 3.6 — Выключить проект

В терминале :

```text
Ctrl + C
```

Сказать :

> "Когда вы останавливаете сервер, приложение останавливается. Это нормально. Ничего не сломалось."

### 3.7 — Снова включить проект

Запустить снова :

```bash
npm run dev
```

И заставить студентов прямо вживую повторить этот цикл :

1. включить
2. открыть `localhost`
3. выключить
4. снова включить

**Это повторение критически важно.**
Именно оно убирает впечатление, что современный проект — это что-то страшное и хрупкое.

## ЧАСТЬ 3 — Читаем минимальную структуру проекта (5 мин)

### 4.1 — Что показать, а что не трогать

Не нужно пытаться объяснить всю сгенерированную структуру.
Нужно показать только то, что полезно сегодня :

- `app`
- `app/page.tsx`
- `public`
- `components` (мы создадим эту папку сами)

### 4.2 — Карта в голове, которую нужно дать

В App Router папки и файлы задают маршруты.

Пример :

```text
app/page.tsx -> /
app/offers/page.tsx -> /offers
app/contact/page.tsx -> /contact
```

Эта идея должна стать автоматической :

> "Я создаю папку внутри `app`, потом файл `page.tsx`, и так появляется новая страница."

### 4.3 — Что означает `components`

Папка `components` — это не маршрут.

Она нужна для хранения переиспользуемых блоков :

- карточек
- меню
- кнопок
- секций

**Ключевая мысль** :

> "Страницу посещают. Компонент используют."

## ЧАСТЬ 4 — Практика с сопровождением : страницы, компонент, навигация (45 мин)

### 5.1 — Цель упражнения

В конце у каждого студента должно получиться примерно вот это :

```text
pilote-business-hub/
├── app/
│   ├── page.tsx
│   ├── offers/
│   │   └── page.tsx
│   ├── contact/
│   │   └── page.tsx
│   └── layout.tsx
└── components/
    └── OfferCard.tsx
```

И в браузере :

- главная страница
- страница предложений
- страница контакта
- навигация, которая позволяет переходить между ними

### 5.2 — Логика этого упражнения

Мы не пытаемся сделать целый сайт.
Мы пытаемся поставить 4 рефлекса :

1. создать файл в правильном месте
2. понять, что папка внутри `app` становится маршрутом
3. переиспользовать компонент
4. сразу проверять результат в браузере

### 5.3 — Шаг 1 : упростить главную страницу

Попросить ИИ очень конкретно, например так :

> "В моем проекте Next.js упрости `app/page.tsx`. Мне нужна очень простая главная страница с заголовком, абзацем и ссылкой на `/offers`. Не трогай другие файлы."

**Почему это хорошая формулировка** :

- она называет конкретный файл
- она просит одно точное действие
- она ограничивает поле изменений

### 5.4 — Шаг 2 : создать страницу `/offers`

Попросить :

> "Создай страницу `app/offers/page.tsx` с заголовком, коротким текстом и ссылкой на `/contact`. Код должен оставаться простым."

Потом проверить в браузере :

```text
http://localhost:3000/offers
```

### 5.5 — Шаг 3 : создать страницу `/contact`

Попросить :

> "Создай страницу `app/contact/page.tsx` с заголовком, коротким текстом и простым визуальным блоком формы. Настоящую логику пока не добавляй."

Потом проверить :

```text
http://localhost:3000/contact
```

### 5.6 — Шаг 4 : создать переиспользуемый компонент

Нужно объяснить :

> "Теперь мы перестаем повторять один и тот же кусок кода. Мы создаем переиспользуемую деталь."

Можно сделать простой компонент, например `OfferCard.tsx`.

Попросить :

> "Создай компонент `components/OfferCard.tsx`. Мне нужен простой блок с заголовком и коротким текстом."

Потом попросить :

> "Используй этот компонент три раза в `app/page.tsx`, чтобы показать три преимущества моего предложения."

**Именно здесь идея компонента становится ощутимой.**

Студенты видят :

- что они не пишут один и тот же блок 3 раза вручную
- что они создают одну деталь
- что потом они ее используют снова и снова

### 5.7 — Шаг 5 : сделать навигацию

Теперь нужно, чтобы по интерфейсу можно было кликать и переходить.

Есть две педагогические опции :

- либо менять `app/layout.tsx` напрямую
- либо создать отдельный компонент `MainNav.tsx` и подключить его в `layout.tsx`

Для этой сессии обычно проще :

- остаться в `app/layout.tsx`
- добавить минимальную навигацию там

Попросить :

> "Добавь в `app/layout.tsx` простую навигацию со ссылками на `/`, `/offers` и `/contact` с помощью `Link`."

Потом попросить студентов кликнуть вживую :

- Главная
- Предложения
- Контакт
- вернуться обратно

### 5.8 — Шаг 6 : проверить, что все работает

Простой чеклист :

- терминал все еще работает
- `localhost` открывается
- `/offers` отвечает
- `/contact` отвечает
- ссылки реально позволяют переходить
- компонент виден на главной

### 5.9 — Что нужно повторять во время практики

Магомед должен постоянно говорить фразы такого типа :

> "Если не работает, это не страшно. Сейчас локализуем проблему."

> "Не меняйте десять вещей одновременно."

> "Проверяйте после каждого маленького изменения."

> "Копируйте точную ошибку. Сначала не пересказывайте ее своими словами."

### 5.10 — Как правильно разговаривать с ИИ во время этой практики

Нужно дать мини-правило :

**Плохой запрос :**

> "Сделай мне весь сайт."

**Хороший запрос :**

**Плохой запрос :**

## ЧАСТЬ 5 — Дебаг : учимся исправлять без паники (10 мин)

### 6.1 — Антипаническая рутина

Нужно дать фиксированный метод из 5 шагов :

1. **Прочитать точную ошибку**
2. **Понять, в какой момент она появляется**
3. **Сначала проверить самое простое**
4. **Попросить минимальное исправление**
5. **Снова запустить и снова проверить**

### 6.2 — Как определять момент проблемы

Студенты должны научиться задавать себе вопросы :

- проблема во время создания проекта ?
- проблема в момент `npm run dev` ?
- проблема уже в браузере ?
- проблема открывается только на одной конкретной странице ?
- проблема началась после правки определенного файла ?

### 6.3 — Первые проверки, которые нужно сделать

Прежде чем называть это "большим багом", нужно проверить :

- терминал все еще открыт ?
- я точно нахожусь в папке проекта ?
- файл был сохранен ?
- название папки написано точно ?
- файл действительно называется `page.tsx` ?
- путь импорта написан правильно ?

### 6.4 — Частые ошибки на С6

| Симптом | Возможная причина | Первый рефлекс |
|----------|------------------|----------------|
| `npx` или `npm` не распознается | Node.js установлен неправильно или терминал не был перезапущен после установки | проверить Node.js и заново открыть терминал |
| `npm run dev` не работает | вы не в той папке или установка завершилась не полностью | проверить папку и попробовать снова |
| браузер показывает, что страница не найдена | папка маршрута или файл лежат не там, где нужно | проверить `app/offers/page.tsx` или `app/contact/page.tsx` |
| компонент не отображается | неправильный импорт или неправильное имя файла | сравнить реальный файл и импорт |
| в браузере ничего не меняется | файл не сохранен или сервер уже остановлен | сохранить файл и посмотреть на терминал |
| ссылка не работает | `Link` не импортирован или маршрут написан неверно | проверить сам текст ссылки и импорт |
| порт `3000` уже занят | другой сервер уже работает | принять другой порт или остановить старый сервер |

### 6.5 — Самая важная мысль

> "Ошибка — это не доказательство того, что вы слабый. Ошибка — это информация."

И еще точнее :

> "Ваша работа не в том, чтобы избежать всех ошибок. Ваша работа — научиться читать сигнал, который они присылают."

## ЧАСТЬ 6 — Анонс общего проекта формации (5 мин)

### 7.1 — Какой проект мы предлагаем

Лучший общий проект для этой формации — не "красивый проект".
Нужен проект, который :

- полезен
- легко адаптируется под разные профили
- может быть монетизирован
- достаточно прост, чтобы стартовать
- достаточно богат, чтобы расти в течение 3 месяцев

### 7.2 — Предложение

**Общий проект формации : мини-приложение для квалификации клиента и записи на встречу.**

Принцип очень простой :

- человек заходит в приложение
- понимает предложение
- проходит небольшой опрос
- оставляет свои данные
- запрашивает звонок, встречу или контакт

Потом в течение формации эту базу можно постепенно усложнять :

- более чистые страницы
- более структурированные компоненты
- настоящая форма
- хранение заявок
- авторизация
- панель управления
- автоматизации
- почтовая или календарная интеграция
- добавление ИИ, если это уместно

### 7.3 — Почему этот проект особенно хорош

1. **Он полезен сразу**
   - его может использовать фрилансер
   - его может использовать коуч
   - его может использовать агентство
   - его можно продать клиенту как услугу

2. **Он потенциально приносит деньги**
   - либо как инструмент для собственного бизнеса
   - либо как сервис для чужого бизнеса

3. **Он идеально подходит педагогически**
   - начинается со страниц
   - продолжается компонентами
   - потом переходит к опросам и формам
   - потом к данным
   - потом к авторизации
   - потом к более сложной логике

4. **Каждый может сделать его своим**
   - коуч : квалификация перед вводным звонком
   - консультант : мини-аудит перед предложением
   - агентство : сбор брифа
   - недвижимость : квалификация продавца или покупателя
   - локальная услуга : запрос на оценку

### 7.4 — Как объявлять это без лишнего страха

Не нужно говорить :

> "Вы будете строить большой сложный SaaS."

Нужно говорить :

> "Мы все стартуем с одной простой базы : приложение, которое показывает предложение, собирает потребность и получает контакт. А потом мы будем усиливать его сессия за сессией."

### 7.5 — Как это связано с практикой сегодняшнего дня

Красота этого выбора в том, что практика С6 **уже является началом проекта** :

- главная страница
- страница предложений
- страница контакта
- навигация

То есть студенты делают не одноразовое упражнение.
Они уже закладывают первые стены общего проекта.

## ДЕБРИФ + ДОМАШКА

### 5 идей, которые нужно унести с собой

1. **Next.js не убирает JavaScript, а организует его**
2. **Страница соответствует маршруту**
3. **Компонент — это переиспользуемый блок**
4. **Включать и выключать проект — нормальная часть работы**
5. **Ошибки — это часть практики, и их можно спокойно исправлять**

### 3-2-1

- **3 вещи, которые вы теперь умеете**
- **2 места, где вы еще буксуете**
- **1 действие, которое вы повторите сегодня вечером или завтра**

### Рекомендуемая домашка

Домашка должна остаться простой и полезной :

1. самостоятельно снова включить проект
2. выключить его и снова включить второй раз
3. добавить четвертую страницу, связанную с вашей реальной деятельностью
4. переписать тексты главной страницы под свой рынок
5. зафиксировать одну ошибку, с которой вы столкнулись, и как именно вы ее исправили

### Что здесь нужно оценивать

Не :

- красоту дизайна
- идеальность кода
- скорость работы

А :

- проект запускается ?
- в нем несколько страниц ?
- между ними можно переходить ?
- студент прошел хотя бы через одну проблему и не убежал от нее ?

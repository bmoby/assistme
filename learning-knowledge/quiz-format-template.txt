================================================================
ШАБЛОН ФОРМАТА КВИЗОВ v2
================================================================

Этот файл определяет формат квизов для Discord-бота.
Админ загружает этот файл через /quiz-create, Claude парсит
его в структурированный JSON, админ проверяет превью и
подтверждает отправку студентам.

Язык контента: русский
Язык вопросов и ответов: русский

================================================================
СТРУКТУРА ФАЙЛА
================================================================

SESSION: [номер]
TITLE: [название сессии]

Вопросы идут последовательно, разделены --- (три дефиса).
Блоки (BLOCK) необязательны, просто для организации.

================================================================
ТИПЫ ВОПРОСОВ (3 типа)
================================================================

--------------------------------------------------------------
[QCM] — Выбор ответа (один правильный)
--------------------------------------------------------------

[QCM]
Q[N]. [Текст вопроса]

   A) [Вариант A]
   B) [Вариант B]
   C) [Вариант C]
   D) [Вариант D]

ANSWER: [буква]
EXPLANATION: [объяснение, необязательно]

Правила:
- Всегда 4 варианта: A, B, C, D
- ANSWER = только буква (A, B, C или D)
- Неправильные варианты должны быть правдоподобными
- EXPLANATION показывается в обратной связи если студент ошибся

--------------------------------------------------------------
[VF] — Правда или Ложь
--------------------------------------------------------------

[VF]
Q[N]. [Утверждение]

ANSWER: true | false
EXPLANATION: [объяснение, необязательно]

Правила:
- Студент видит две кнопки: «Правда» и «Ложь»
- ANSWER = true или false

--------------------------------------------------------------
[OPEN] — Открытый вопрос (свободный ответ)
--------------------------------------------------------------

[OPEN]
Q[N]. [Текст вопроса]

ANSWER: [эталонный ответ или ключевые моменты через ;]
EXPLANATION: [объяснение, необязательно]

Правила:
- Студент пишет ответ свободно в чат
- ANSWER = ориентир для ИИ-проверки (не дословный ответ)
- Проверка ОЧЕНЬ гибкая: синонимы, перефразировки,
  аналогии и объяснения своими словами принимаются
- Несколько ключевых моментов разделяются через ;

ПОЛНЫЙ ПРИМЕР

SESSION: 2
TITLE: Цифровой мир

================================================================
BLOCK 1 — Фундамент
================================================================

[QCM]
Q1. Вы попросили ИИ создать интернет-магазин. Работает!
    Потом добавили оплату, личный кабинет, чат.
    Через месяц всё тормозит и падает. Почему?

    A) ИИ плохо написал код
    B) Основа сайта не была рассчитана на столько функций
    C) Нужен более мощный компьютер
    D) Слишком много посетителей

ANSWER: B
EXPLANATION: Без продуманной архитектуры каждая новая функция создаёт нагрузку на слабый фундамент.

---

[VF]
Q2. API — это программа, которая работает только на компьютере пользователя.

ANSWER: false
EXPLANATION: API — это интерфейс между клиентом и сервером, он работает на стороне сервера.

---

[OPEN]
Q3. Объясните своими словами: почему нельзя добавить
    безопасность в проект «потом», после запуска?

ANSWER: Безопасность должна быть заложена в архитектуру с самого начала; добавление позже требует переписывания основ; данные могут быть скомпрометированы до добавления защиты
EXPLANATION: Безопасность — это часть фундамента, а не надстройка.

---

================================================================
BLOCK 2 — Ресторан
================================================================

[QCM]
Q4. Вы ставите лайк в Instagram. Что происходит?

    A) Клиент записывает лайк прямо в холодильник (базу)
    B) Клиент передаёт через API, сервер сохраняет лайк в базе
    C) Клиент идёт напрямую на кухню
    D) Лайк остаётся только на вашем телефоне

ANSWER: B

---

[OPEN]
Q5. Почему API важен? Что было бы без него?

ANSWER: API контролирует что можно запросить и как; без него клиент общался бы с сервером напрямую; хаос и отсутствие безопасности

---

[VF]
Q6. Сервер — это компьютер, который всегда включён и ждёт запросов от клиентов.

ANSWER: true
EXPLANATION: Именно так — сервер работает 24/7 и обрабатывает запросы от клиентов через API.

---

[QCM]
Q7. Друг говорит: «Хочу сделать приложение как Uber —
    просто нужен красивый дизайн.» Что не так?

    A) Дизайн не важен, нужно только кодить
    B) Дизайн — это только клиент. За ним нужен сервер, база данных и API
    C) Uber не использует серверы
    D) Достаточно нанять дизайнера

ANSWER: B
EXPLANATION: Uber — это сложная система с геолокацией, платежами, базой данных. Дизайн — лишь верхушка айсберга.

---

================================================================
СОВЕТЫ ПО СОСТАВЛЕНИЮ
================================================================

1. Чередуйте типы вопросов (не только QCM или только OPEN)
2. Вопросы OPEN проверяют понимание, а не память
3. ANSWER для OPEN = ориентир для проверки, не точный ответ
4. Неправильные варианты QCM должны быть правдоподобными
5. Сложность должна соответствовать уровню сессии
6. EXPLANATION необязателен, но рекомендуется — показывается
   студенту в обратной связи при неправильном ответе
7. Содержание должно быть в рамках темы сессии

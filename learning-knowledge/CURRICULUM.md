# Формация "Pilote Neuro" Сессия 2 — Учебная программа

> **Статус: В ПОДГОТОВКЕ**

**Цель** : Научить нетехнарей проектировать любую цифровую систему с помощью ИИ.
**Формат** : 3 месяца, 2 сессии по 2ч/неделю + задания + документы
**Студенты** : максимум 30, начинающие с мотивацией
**Цена** : 1200€ (4 платежа по 300€)

---

## Видение формации

Формация НЕ учит программированию. Она передаёт:

1. **Понимание** — Как работает код (не синтаксис, а логика)
2. **Архитектура** — Как проект декомпозируется и структурируется
3. **Использование ИИ** — Как общаться с Claude, ChatGPT и др. для создания продуктов
4. **Критическое мышление** — Проверять, тестировать, ставить под вопрос работу ИИ
5. **Дизайн и эргономика** — Объективные правила (не вкусы), mobile-first, UX
6. **Mindset** — ИИ ускоряет, но не упрощает. Когда все быстрые, нужно делать ЛУЧШЕ.

## Уроки Сессии 1

### Что сработало
- Качество преподавания (высокая мотивация)
- Лайв-сессии очень ценились
- 14 записалось → 10 закончили → 6 работают в команде

### Что не сработало (логистика)
1. **Google Drive** → постоянные проблемы с правами доступа
2. **Запись лайвов** → 100% ручной процесс
3. **PDF-конспекты** → никогда не хватало времени
4. **Задания** → все сдавали в последний день → перегрузка проверки

### Решения для Сессии 2
- Supabase Storage заменяет Google Drive (ноль проблем с доступом)
- Структурированная система сдачи с поэтапными дедлайнами
- Пре-ревью ИИ заданий (снижает 80% нагрузки)
- Автоматические уведомления (дедлайны, напоминания, новый контент)
- Discord-бот автоматизирует логистику

## Педагогические принципы

1. **Конкретное перед абстрактным** — Студент проживает проблему (1 сессия контролируемого хаоса), затем получает метод как желанное решение. Никогда теории без предварительного опыта.
2. **Spec-Driven Development** — Спек — источник истины. Преподаётся ПОСЛЕ того, как студент пережил хаос — цикл хаос→откровение за 4 сессии максимум.
3. **Проект — красная нить** — Каждый студент строит СВОЙ личный проект. Вся теория служит проекту.
4. **Just-in-time tooling** — Каждый инструмент ИИ (config, skills, agents) вводится В МОМЕНТ когда студенту он нужен, не раньше.
5. **Человек делает то, что ИИ не может** — Env vars, API keys, регистрации, аккаунты, валидация. Незаменимая роль пилота.
6. **Последовательный scaffolding** — С проводником (М1) → полуоткрытый (М2-М3) → свободный (М4-М5). Никогда назад.
7. **Flipped classroom** — Короткое видео (10-15 мин) ДО сессии. Лайв = 100% практика.
8. **Peer review СПЕКОВ** — Оценивать спеки других, находить слабости, предлагать альтернативы.
9. **Предупреждать долину** — Назвать долину с С1. Запланированные интервенции на неделях 5-6.
10. **Артефакт на каждой сессии** — Каждая сессия производит что-то конкретное (URL, схема, спек, прототип). Никогда чисто документальных сессий.

## Система анти-отчисления

**4 критических момента отчисления:**
1. **Недели 1-2** : Плохое совпадение, технические проблемы, изоляция → Крепкий пре-бординг + индивидуальный контакт в первые 72ч
2. **Недели 4-5** : Долина отчаяния, новизна исчезла → Церемония середины, лёгкое и приятное задание
3. **Недели 7-8** : Усталость середины пути → Облегчённая нагрузка заданий, более лёгкие сессии
4. **Недели 10-11** : Стена проекта → Обязательные контрольные точки на неделях 9-10 для раннего обнаружения

**Показатели для отслеживания по студенту:**
- Присутствие на лайвах
- Активность в Discord
- Сдача заданий
- Участие в еженедельных ритуалах

**Правило** : 2 красных показателя = контакт в течение 48ч. 3+ = личный голосовой звонок.

## Учебная программа — 12 недель, 24 сессии

### Нить прогрессии

```
Недели 1-2  : ОТКРЫТЬ            → "Я понимаю цифровой мир и умею пользоваться ИИ"
Недели 3-4  : МЕТОД              → 1ч хаоса → откровение → спек → rebuild (полный цикл)
Недели 5-6  : АРСЕНАЛ ИИ        → Config, skills, agents — каждый инструмент когда нужен + церемония
Недели 7-8  : СТРОИТЬ            → Структурированные спринты со спеком + полный арсенал
Недели 9-10 : ПРОФЕССИОНАЛИЗАЦИЯ → Предложение клиенту, продвинутый peer review, MVP
Недели 11-12: СДАТЬ              → Тесты, деплой, demo day, выпуск
```

**Психологический принцип** : хаос длится 1 сессию (С5), откровение приходит в С6 (48ч позже). Контраст мощный, но фрустрация короткая. Каждая сессия производит конкретный артефакт.

**Блум** : Понимание + Применение (базовое)

**Сессия 1 (День 1) — Кик-офф + Quick Win**

*Без видео перед сессией (день 1)*

> **Артефакт** : URL онлайн, которым можно поделиться.

- **Церемония Кик-офф (50 мин)**

**Сессия 2 — Цифровой мир: зал и кухня**

*Видео перед сессией (10 мин) : "Ваш сайт — это просто ресторан — часть 1"*

> **Артефакт** : схема архитектуры известного приложения (зал + кухня + рецепты).

- **Аналогия ресторана — часть 1**
  - Зал = front-end | Кухня = back-end | Меню = интерфейс | Рецепт = алгоритм
  - **Ключевой концепт** : разделение front/back — пользователь никогда не видит кухню
  - Демо : DevTools на известном сайте

- **Декомпозиция (первый подход)**
  - Блиц-упражнение : алгоритм утренней рутины
  - Условия ЕСЛИ/ИНАЧЕ : "ЕСЛИ холодильник пустой ТОГДА..."
  - Псевдокод : писать логику на обычном языке

- **Задание (45 мин)** : выбрать приложение (Uber, Instagram), нарисовать его ресторан (зал + кухня + рецепты).

**Сессия 3 — Цифровой мир: холодильник, поставщики и охранник**

*Видео перед сессией (10 мин) : "Ваш сайт — это просто ресторан — часть 2"*

> **Артефакт** : ПОЛНАЯ схема архитектуры того же приложения.

- **Аналогия ресторана — часть 2 (полная система)**
  - Холодильник = база данных (таблица с суперспособностями)
  - Официант = API (связь зал-кухня)
  - Поставщики = внешние API (сторонние сервисы)
  - Бейдж персонала = аутентификация
  - Адрес = DNS
  - **Точка разрыва** : веб-сервер обслуживает 1000 клиентов параллельно

- **Паттерны** : сравнить 3 похожих приложения — "у них всех одинаковый ресторан"

- **Задание (45 мин)** : дополнить схему С2 новыми элементами. Peer review.

**Spacing** : концепты С2 ПОВТОРЯЮТСЯ в С3 (то же приложение, обогащённая схема = повторение через 5 дней)

**Сессия 4 — Ландшафт ИИ 2026**

*Видео перед сессией (15 мин) : "Claude, Cursor, Codex — в чём разница?"*

> **Артефакт** : страница, построенная в ИИ-редакторе (не генераторе).

- **3 категории инструментов ИИ**
  1. **Генераторы (v0, Lovable, Bolt)** : просто, мало контроля (= С1)
  2. **ИИ-редакторы (Cursor, Windsurf)** : итеративный контроль, больше точности
  3. **Ассистенты кода (Claude Code, Codex CLI)** : продвинутое управление, самый мощный

- **"Мы прогрессируем от генератора к ассистенту кода"**

- **Демо** : тот же мини-проект в 3 инструментах — увидеть разницу

- **Задание (45 мин)** : построить в Cursor или Claude Code. Декомпозированный бриф (С2-С3) → ИИ → результат → итерация.

**Конец Модуля 1** : студент понимает цифровой мир, умеет декомпозировать, знает инструменты ИИ. 2 вещи онлайн. Готов строить.

### МОДУЛЬ 2 — Метод (Недели 3-4)

**Цель модуля** : Студент проживает 1 сессию контролируемого хаоса, затем открывает метод spec-driven как решение. Пишет свой спек, методично управляет ИИ, и получает peer review. Цикл хаос→откровение ЗАВЕРШЁН за 4 сессии.

**Блум** : Анализ + Применение
**Scaffolding** : Полуоткрытый (рамка дана, студент делает выбор)

**С5 : Анонсировать долину.** "Через 2-3 недели вам захочется бросить. Это запланировано."

**Сессия 5 — Контролируемый хаос (1 сессия, не 4)**

*Видео перед сессией (10 мин) : "Как выбрать проект, который будет мотивировать вас 10 недель"*

> **Артефакт** : кривой, но рабочий прототип — И список фрустраций.

- **Выбор проекта — красной нити (20 мин)**
  - Критерии : личный, scop-абельный, технически разнообразный
  - 5-8 предложенных шаблонов : бот Telegram, приложение управления, лендинг + CRM, дашборд, портфолио
  - "Выберите то, что вас заводит"

- **Фаза 1 : "Вперёд" (40 мин)**
  - Лёгкое указание : "Опишите свой проект ИИ и начинайте строить. Без навязанного метода."
  - Студенты бросаются. Разные результаты. Магомед наблюдает, помогает технически, но НЕ даёт метод.

- **Фаза 2 : "Добавьте фичу" (30 мин)**
  - Указание : "Теперь добавьте [функцию, требующую БД или API]"
  - Тут ломается. Студент не предусмотрел структуру данных. ИИ не понимает контекст. Нужно возвращаться.

- **Немедленный дебриф (20 мин)** — НЕ давать фрустрации затянуться
  - "Что сработало? Что сломалось? Чего вам НЕ ХВАТИЛО?"
  - Записать фрустрации в Discord `#хаос-отзывы`
  - Магомед : "Всё, что вы только что пережили — это будни 80% ИИ-проектов. В следующей сессии я покажу метод, который решает ВСЁ это."

**Психология** : хаос длится 70 минут, не 2 недели. Фрустрация острая, но короткая. Тизер решения создаёт предвкушение.

**Сессия 6 — Откровение (48ч после хаоса)**

*Видео перед сессией (12 мин) : "Почему 80% ИИ-проектов проваливаются — и метод, который всё меняет"*

> **Артефакт** : шаги 1-3 метода применённые к своему проекту (описание + исследование + вопросы). Конкретный документ, не теория.

- **Открытие (10 мин)** : вернуться к фрустрациям из `#хаос-отзывы`
  - "Расплывчатый бриф → расплывчатый результат" = проблема СПЕЦИФИКАЦИИ
  - "Я забыл сказать что..." = проблема ИССЛЕДОВАНИЯ
  - "Не помню где остановился" = проблема ROADMAP
  - "Не знаю, хорошо ли получилось" = проблема ВАЛИДАЦИИ

- **Метод в 6 шагов (25 мин)** :
  1. **ОПИСАТЬ** : всё из головы на бумагу → конец расплывчатым брифам
  2. **ИССЛЕДОВАТЬ** : ИИ проводит глубокое исследование → конец слепым зонам
  3. **СПРАШИВАТЬ** : ИИ задаёт вопросы + "ПОЧЕМУ этот вопрос важен" → конец забытому
  4. **ПОНЯТЬ** : прочитать вопросы, найти ответы → конец "я не знал"
  5. **ПРИОРИТИЗИРОВАТЬ** : срочно vs может подождать, MVP vs V2 → конец параличу
  6. **СПЕЦИФИЦИРОВАТЬ** : формальный документ = источник истины → конец "где я остановился?"

  Каждый шаг иллюстрируется контрастом с опытом С5

- **Задание (70 мин)** : применить шаги 1-3 к СВОЕМУ проекту
  - Шаг 1 : написать полное описание
  - Шаг 2 : запустить исследование с ИИ
  - Шаг 3 : сгенерировать вопросы + "почему каждый вопрос полезен"
  - **Ключевой момент** : студент осознаёт всё, что забыл в С5

---

**Сессия 7 — Написать спек + первый файл конфигурации**

*Видео перед сессией (10 мин) : "Анатомия спека, который работает"*

> **Артефакт** : полный спек проекта + файл CLAUDE.md или .cursorrules (первый инструмент интегрированный в метод).

- **Написать спек (50 мин)** : завершить шаги 4-6
  - Шаблон : обзор, архитектура, схема данных, функции, бизнес-правила, roadmap задач
  - Каждая секция отвечает на потребность, пережитую в С5

- **Первый ИИ-инструмент : файл конфигурации (30 мин)** — just-in-time
  - "У вас есть спек. Теперь ДАЙТЕ его ИИ как постоянный контекст."
  - Создать `CLAUDE.md` или `.cursorrules` из спека
  - Тест : тот же вопрос с и без конфига → увидеть разницу
  - "Вы больше никогда не будете заново вводить контекст проекта."

- **Задание (30 мин)** : доработать спек + конфиг, оба вместе

---

**Сессия 8 — Управлять ИИ со спеком + peer review**

*Видео перед сессией (10 мин) : "Как дать спек ИИ, чтобы он идеально выполнял"*

> **Артефакт** : первые задачи выполнены по методу + спек улучшен пирами.

- **Что делает ИИ vs что делает человек (10 мин)**

  | ИИ делает | Человек делает |
  |-----------|----------------|
  | Генерирует код | Пишет спеки |
  | Предлагает архитектуру | Валидирует выбор |
  | Дебажит | Решает что исправлять |

  Что ИИ НЕ МОЖЕТ: создавать аккаунты, генерировать API ключи, инжектить `.env`, тестировать с реальными пользователями, решить хорош ли результат.

- **Спринт spec-driven (50 мин)** : выполнить 3-4 задачи из roadmap
  - Workflow : задача → контекст из спека → ИИ генерирует → прочитать резюме → тестировать → roadmap (ГОТОВО)
  - Создать нужные аккаунты + получить API ключи + `.env`
  - **Вывод** : быстрее, точнее, и всегда знаешь где остановился vs С5

- **Peer review спеков (50 мин)**
  - Каждый получает спек другого студента
  - Метод : прочитать → ИИ ищет слабости → личный анализ → предложения по улучшению
  - Презентация возвратов + обсуждение + обновление спеков

- **Checkpoint** : самооценка (понимание, мотивация, качество спека)

**Конец Модуля 2** : за 4 сессии студент пережил хаос (С5), открыл метод (С6), написал спек + конфиг (С7), управляет ИИ и получил ревью (С8). Полный цикл.

**Сессия 9 — Skills и переиспользуемые промпты**

*Видео перед сессией (10 мин) : "Skills: ваш постоянный набор инструментов"*

> **Артефакт** : 4 скилла настроены и протестированы на коде проекта.

- **Проблема (пережитая в М2)** : повторяем одни и те же инструкции 50 раз. "Проверь эргономику", "Объясни что ты сделал" → это должно быть автоматическим.

- **Skills / переиспользуемые промпты**
  - Skill = предварительно написанная инструкция, вызываемая в любой момент
  - Claude Code : `.claude/commands/` | Cursor : prompt library
  - Написать ОДИН раз, использовать бесконечно

- **4 основных скилла** :
  1. **Explainer** : список изменённых файлов + объяснение простым языком
  2. **UX Review** : критика интерфейса (иерархия, контраст, отступы, согласованность, mobile-first)
  3. **Spec Sync** : сравнение код vs спек (реализовано, отсутствует, расхождение)
  4. **Pre-commit Review** : крайние случаи, безопасность, читаемость

- **Задание (60 мин)** : создать 4 скилла, протестировать на своём коде
  - Спринт : продвинуть 2-3 задачи roadmap С активными скиллами

- **Интервенция долины** : личное голосовое сообщение от Магомеда на этой неделе

**Сессия 15 — Дебаг и критическое мышление**

*Видео перед сессией (10 мин) : "ИИ иногда врёт — как это обнаружить"*

> **Артефакт** : найденные и исправленные баги + персональный чеклист ревью.

- **Почему эта сессия именно сейчас** : проект достаточно продвинут, чтобы иметь настоящие баги. Критическое мышление приходит когда есть что критиковать.

- **Think-Aloud Protocol (20 мин)** : Магомед ревьюит код в прямом эфире, озвучивая мысли
  - "Вижу 'if user' но если user — null?"
  - "Эта функция делает 3 вещи — сигнал тревоги"
  - Студент учится ПРОЦЕССУ, не синтаксису

- **4 вопроса ревью** (простой чеклист) :
  1. "Что произойдёт, если пользователь сделает неожиданное?"
  2. "Что произойдёт, если сеть медленная или оборвалась?"
  3. "Чувствительные данные защищены?"
  4. "Код делает ОДНУ вещь или слишком много?"

- **Задание (60 мин)** : применить 4 вопроса к своему проекту + исправить
  - Затем : ИИ генерирует код с намеренными багами — студент их находит

- **Спринт polish (30 мин)** : состояния загрузки, понятные сообщения об ошибках, адаптивность
